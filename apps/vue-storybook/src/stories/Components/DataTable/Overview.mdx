# DataTable

A stateless, controlled-component data table designed for server-side data fetching. Sorting, filtering, and pagination state flow through `v-model:*` bindings; the parent owns state and triggers data fetches.

## Features

- **Stateless data axes**: Parent owns `sorting`, `filters`, `pagination` via v-model — no duplicated internal state.
- **Per-feature opt-in**: `enableSorting`, `enableFilter`, `enablePagination` independently control whether DataTable renders the corresponding UI.
- **Identical components inside or outside**: The same `<Filters>` and `<Pagination>` composites render in DataTable's toolbar/footer or anywhere else in the parent.
- **Composable helper**: `useDataTableController` bundles the three refs, encapsulates the `flush: 'sync'` page-reset rule, and exposes a single merged `state` computed.
- **Visual concerns stay internal**: Row selection, expansion, column visibility / pinning / sizing remain managed inside DataTable.

## Installation

```bash
pnpm add @meldui/vue @tanstack/vue-table
```

## Recommended setup (with composable)

```vue
<script setup lang="ts">
import { computed, ref, watch, h } from 'vue'
import { DataTable, DataTableColumnHeader, useDataTableController } from '@meldui/vue'
import type { ColumnDef } from '@tanstack/vue-table'

interface User {
  id: string
  name: string
  email: string
}

const columns: ColumnDef<User>[] = [
  {
    accessorKey: 'name',
    header: ({ column }) => h(DataTableColumnHeader, { column, title: 'Name' }),
  },
  {
    accessorKey: 'email',
    header: ({ column }) => h(DataTableColumnHeader, { column, title: 'Email' }),
  },
]

const data = ref<User[]>([])
const pageCount = ref(1)
const totalRows = ref(0)

const { sorting, filters, pagination, state } = useDataTableController({
  pageSize: 20,
})

async function fetchPage() {
  const res = await api.listUsers({
    sort: state.value.sorting,
    filters: state.value.filters,
    page: state.value.pagination.pageIndex + 1,
    perPage: state.value.pagination.pageSize,
  })
  data.value = res.data
  pageCount.value = res.meta.total_pages
  totalRows.value = res.meta.total
}

watch(state, fetchPage, { deep: true })
fetchPage()
</script>

<template>
  <DataTable
    :columns="columns"
    :data="data"
    :page-count="pageCount"
    :total-rows="totalRows"
    enable-sorting
    enable-filter
    enable-pagination
    v-model:sorting="sorting"
    v-model:filters="filters"
    v-model:pagination="pagination"
  />
</template>
```

## Emits

`<DataTable>` emits three per-axis v-model events:

- `update:sorting` — payload `SortingState`
- `update:filters` — payload `DataTableFilterState`
- `update:pagination` — payload `PaginationState`

Plus:

- `@retry` — fired when the user clicks Retry in the error state.
- `@refresh` — fired when the toolbar refresh button is clicked.
- `@rowActivate` — fired by keyboard navigation (Enter/Space).

## useDataTableController

Parent-side composable bundling the three v-model refs, the page-reset rule, and a merged `state` computed.

```ts
const { sorting, filters, pagination, state, reset } = useDataTableController({
  pageSize: 20,
  // Optional:
  initialSorting: [{ id: 'name', desc: false }],
  initialFilters: { role: 'admin' },
  initialPagination: { pageIndex: 0, pageSize: 20 },
  resetPageOnFilterChange: true, // default
  resetPageOnSortChange: true, // default
})

watch(state, fetchPage, { deep: true })
```

The composable applies `flush: 'sync'` page-reset watchers internally so a single user action produces exactly one fetch. Without it, a filter change would fire `fetchPage` twice (once before the reset, once after).

## Migration from v1

| v1                                                        | v2                                                                              |
| --------------------------------------------------------- | ------------------------------------------------------------------------------- |
| `onServerSideChange`                                      | `update:sorting` / `update:filters` / `update:pagination`                       |
| `initialFilters` / `initialSorting` / `initialPagination` | `useDataTableController({ initialFilters, initialSorting, initialPagination })` |
| `showPagination`                                          | `enablePagination` (default **false**)                                          |
| `searchColumn` / `searchPlaceholder`                      | `filterSearch={ id, placeholder }`                                              |
| `DataTablePagination`                                     | `Pagination` (standalone composite)                                             |

Pagination now resets to page 0 on **sort change** in addition to filter change (industry-aligned). Opt-out via `useDataTableController({ resetPageOnSortChange: false })`.
