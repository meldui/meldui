# Color System

MeldUI uses a comprehensive color system built with CSS variables and OKLCH color space for consistent, accessible colors across light and dark modes.

## Color Philosophy

- **OKLCH Color Space**: Perceptually uniform colors that look consistent across different hues
- **CSS Variables**: Easy theming and customization
- **Automatic Dark Mode**: All colors have dark mode variants
- **Semantic Naming**: Colors convey meaning (success, warning, destructive, etc.)

## Usage

### In CSS

```css
.my-component {
  background-color: var(--primary);
  color: var(--primary-foreground);
  border: 1px solid var(--border);
}
```

### With Tailwind

```vue
<template>
  <div class="bg-primary text-primary-foreground border border-border">
    Primary colored content
  </div>
</template>
```

### Customizing Colors

Override any color by defining it in your app's CSS:

```css
:root {
  --primary: oklch(0.6 0.15 275); /* Custom purple */
  --destructive: oklch(0.577 0.245 27.325); /* Custom red */
}

.dark {
  --primary: oklch(0.7 0.15 275); /* Lighter purple in dark mode */
}
```

## Dark Mode

All colors automatically adapt to dark mode when the `dark` class is applied to the root element:

```vue
<template>
  <div class="dark">
    <!-- All colors switch to dark mode variants -->
  </div>
</template>
```

## Accessibility

- All color combinations meet WCAG 2.1 AA contrast requirements
- Semantic colors provide clear meaning
- OKLCH color space ensures perceptually uniform colors
