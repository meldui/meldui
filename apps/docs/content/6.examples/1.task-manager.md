The Task Manager is a complete application built entirely with MeldUI components. It lives in the monorepo at `apps/task-manager/` and demonstrates how to compose MeldUI's base components, composite components, and icon packages into a production-style app with routing, state management, and form validation.

## Architecture Overview

The application is a single-page Vue 3 app using Vue Router for navigation and reactive stores for state. All data is persisted in localStorage so the app works without a backend.

## Sidebar Navigation

The app uses the MeldUI Sidebar component for its primary navigation. Each menu item is a `SidebarMenuButton` with an icon and label. The sidebar supports collapsing to icon-only mode via the `collapsible="icon"` prop, and tooltips appear when collapsed to show the item name.

```vue
<script setup lang="ts">
import { IconChecklist, IconFolders, IconLayoutDashboard, IconSettings } from '@meldui/tabler-vue'
import {
  Sidebar,
  SidebarContent,
  SidebarFooter,
  SidebarGroup,
  SidebarGroupContent,
  SidebarGroupLabel,
  SidebarHeader,
  SidebarMenu,
  SidebarMenuButton,
  SidebarMenuItem,
  SidebarSeparator,
  Tooltip,
  TooltipContent,
  TooltipProvider,
  TooltipTrigger,
} from '@meldui/vue'
import { useRoute, useRouter } from 'vue-router'

const route = useRoute()
const router = useRouter()

const navigation = [
  { name: 'Dashboard', path: '/dashboard', icon: IconLayoutDashboard },
  { name: 'Tasks', path: '/tasks', icon: IconChecklist },
  { name: 'Projects', path: '/projects', icon: IconFolders },
]

const isActive = (path: string) => route.path.startsWith(path)
</script>

<template>
  <Sidebar collapsible="icon">
    <SidebarHeader class="p-4">
      <div class="flex items-center gap-2">
        <div
          class="flex h-8 w-8 items-center justify-center rounded-lg bg-primary text-primary-foreground"
        >
          <IconChecklist :size="18" />
        </div>
        <span class="font-semibold group-data-[collapsible=icon]:hidden">Task Manager</span>
      </div>
    </SidebarHeader>
    <SidebarSeparator />
    <SidebarContent>
      <SidebarGroup>
        <SidebarGroupLabel>Navigation</SidebarGroupLabel>
        <SidebarGroupContent>
          <SidebarMenu>
            <SidebarMenuItem v-for="item in navigation" :key="item.path">
              <TooltipProvider>
                <Tooltip>
                  <TooltipTrigger as-child>
                    <SidebarMenuButton
                      :is-active="isActive(item.path)"
                      @click="router.push(item.path)"
                    >
                      <component :is="item.icon" />
                      <span>{{ item.name }}</span>
                    </SidebarMenuButton>
                  </TooltipTrigger>
                  <TooltipContent side="right" class="group-data-[collapsible=icon]:block hidden">
                    {{ item.name }}
                  </TooltipContent>
                </Tooltip>
              </TooltipProvider>
            </SidebarMenuItem>
          </SidebarMenu>
        </SidebarGroupContent>
      </SidebarGroup>
    </SidebarContent>
    <SidebarFooter>
      <SidebarMenu>
        <SidebarMenuItem>
          <SidebarMenuButton :is-active="isActive('/settings')" @click="router.push('/settings')">
            <IconSettings />
            <span>Settings</span>
          </SidebarMenuButton>
        </SidebarMenuItem>
      </SidebarMenu>
    </SidebarFooter>
  </Sidebar>
</template>
```

## Dashboard Page

The Dashboard page composes three building blocks: a stats grid, a task distribution chart, and a recent activity feed. It follows the dashboard block pattern described in the Blocks section.

The stats grid uses a reusable `StatsCard` component that accepts a title, value, icon, optional progress bar, and optional trend indicator:

```vue
<script setup lang="ts">
import { Card, CardContent, CardHeader, CardTitle, Progress } from '@meldui/vue'
import type { Component } from 'vue'

interface Props {
  title: string
  value: string | number
  description?: string
  icon: Component
  progress?: number
  trend?: { value: number; isPositive: boolean }
}

defineProps<Props>()
</script>

<template>
  <Card>
    <CardHeader class="flex flex-row items-center justify-between space-y-0 pb-2">
      <CardTitle class="text-sm font-medium">{{ title }}</CardTitle>
      <component :is="icon" class="h-4 w-4 text-muted-foreground" />
    </CardHeader>
    <CardContent>
      <div class="text-2xl font-bold">{{ value }}</div>
      <p v-if="description" class="text-xs text-muted-foreground">{{ description }}</p>
      <div v-if="trend" class="flex items-center text-xs mt-1">
        <span :class="trend.isPositive ? 'text-green-600' : 'text-red-600'">
          {{ trend.isPositive ? '+' : '' }}{{ trend.value }}%
        </span>
        <span class="text-muted-foreground ml-1">from last week</span>
      </div>
      <Progress v-if="progress !== undefined" :model-value="progress" class="mt-2 h-1" />
    </CardContent>
  </Card>
</template>
```

The Dashboard page assembles these into a responsive grid:

## Task Form with Validation

The task form is the most complex component in the app. It uses vee-validate with Zod for schema-driven validation and integrates several MeldUI components: Input, Textarea, Select, Calendar (via Popover), and TagsInput.

Key patterns demonstrated:

- **Schema validation**: A Zod schema defines all field constraints, which vee-validate enforces automatically.
- **Date picker**: A Calendar inside a Popover, triggered by a Button styled as an input.
- **Tags input**: The TagsInput composite manages an array of string tags with add/remove controls.
- **Two-column grid**: Status and Priority selects sit side by side in a `grid grid-cols-2 gap-4` layout.

```vue
<script setup lang="ts">
import { IconCalendar } from '@meldui/tabler-vue'
import {
  Button,
  Calendar,
  FormControl,
  FormDescription,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
  Input,
  Popover,
  PopoverContent,
  PopoverTrigger,
  Select,
  SelectContent,
  SelectItem,
  SelectTrigger,
  SelectValue,
  TagsInput,
  TagsInputInput,
  TagsInputItem,
  TagsInputItemDelete,
  TagsInputItemText,
  Textarea,
} from '@meldui/vue'
import { toTypedSchema } from '@vee-validate/zod'
import { useForm } from 'vee-validate'
import { z } from 'zod'

const taskSchema = z.object({
  title: z.string().min(1, 'Title is required').max(100),
  description: z.string().max(500).optional(),
  status: z.enum(['todo', 'in-progress', 'done']),
  priority: z.enum(['low', 'medium', 'high']),
  projectId: z.string().nullable(),
  tags: z.array(z.string()),
  dueDate: z.string().nullable(),
})

const { handleSubmit, values, setFieldValue } = useForm({
  validationSchema: toTypedSchema(taskSchema),
  initialValues: {
    title: '',
    description: '',
    status: 'todo',
    priority: 'medium',
    projectId: null,
    tags: [],
    dueDate: null,
  },
})
</script>

<template>
  <form class="space-y-4" @submit="handleSubmit((v) => console.log(v))">
    <FormField v-slot="{ componentField }" name="title">
      <FormItem>
        <FormLabel>Title</FormLabel>
        <FormControl>
          <Input v-bind="componentField" placeholder="Enter task title" />
        </FormControl>
        <FormMessage />
      </FormItem>
    </FormField>

    <div class="grid grid-cols-2 gap-4">
      <FormField v-slot="{ componentField }" name="status">
        <FormItem>
          <FormLabel>Status</FormLabel>
          <Select v-bind="componentField">
            <FormControl>
              <SelectTrigger>
                <SelectValue placeholder="Select status" />
              </SelectTrigger>
            </FormControl>
            <SelectContent>
              <SelectItem value="todo">To Do</SelectItem>
              <SelectItem value="in-progress">In Progress</SelectItem>
              <SelectItem value="done">Done</SelectItem>
            </SelectContent>
          </Select>
          <FormMessage />
        </FormItem>
      </FormField>

      <FormField v-slot="{ componentField }" name="priority">
        <FormItem>
          <FormLabel>Priority</FormLabel>
          <Select v-bind="componentField">
            <FormControl>
              <SelectTrigger>
                <SelectValue placeholder="Select priority" />
              </SelectTrigger>
            </FormControl>
            <SelectContent>
              <SelectItem value="low">Low</SelectItem>
              <SelectItem value="medium">Medium</SelectItem>
              <SelectItem value="high">High</SelectItem>
            </SelectContent>
          </Select>
          <FormMessage />
        </FormItem>
      </FormField>
    </div>

    <div class="flex justify-end gap-2 pt-4">
      <Button type="button" variant="outline">Cancel</Button>
      <Button type="submit">Create Task</Button>
    </div>
  </form>
</template>
```

## Settings Page with Tabs

The Settings page uses a Tabs layout to organize preferences into Appearance, Preferences, and Data sections. The Appearance tab features a visual theme picker using hidden RadioGroupItem inputs with styled Label cards. The Data tab includes export/import functionality and a destructive "Clear All Data" action behind an AlertDialog confirmation.

This mirrors the settings block pattern and demonstrates how to combine Tabs, Card, RadioGroup, Select, Textarea, AlertDialog, and toast notifications into a cohesive settings experience.

## Activity Feed with Timeline

The RecentActivity component shows the five most recently updated tasks using the Timeline composite. Each entry displays a status-colored dot with an icon, the task title, a relative timestamp via `RelativeTime`, and a status Badge. The Timeline component handles all the connector lines and layout automatically.

## Running the App

From the monorepo root:

```bash
# Install dependencies
pnpm install

# Start the task manager dev server
pnpm --filter task-manager dev
```

The app will be available at `http://localhost:5173`. All data is stored in localStorage, so you can create tasks and projects immediately without any backend setup.
