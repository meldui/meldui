## API Reference

### InputOTP

The root component that manages OTP input state.

| Prop         | Type      | Default | Description                           |
| ------------ | --------- | ------- | ------------------------------------- |
| `modelValue` | `string`  | —       | The controlled OTP value (v-model).   |
| `maxLength`  | `number`  | —       | Maximum number of characters.         |
| `disabled`   | `boolean` | `false` | Disable the input.                    |
| `pattern`    | `string`  | —       | Regex pattern for allowed characters. |
| `class`      | `string`  | —       | Custom CSS classes.                   |

#### Events

| Event               | Payload  | Description                         |
| ------------------- | -------- | ----------------------------------- |
| `update:modelValue` | `string` | Emitted when the OTP value changes. |
| `complete`          | `string` | Emitted when all slots are filled.  |

### InputOTPGroup

Groups a set of OTP slots together.

### InputOTPSlot

An individual character slot.

| Prop    | Type     | Default | Description                  |
| ------- | -------- | ------- | ---------------------------- |
| `index` | `number` | —       | The slot index (zero-based). |

### InputOTPSeparator

A visual separator between groups of slots (e.g., a dash).
