# Customization

The DataTable provides multiple customization layers: built-in cell renderers, custom render functions, template slots, row/header styling, and density options.

## Cell Renderers

The `cellRenderers` object provides pre-built render functions for common data formats. Use them in the `cell` option of accessor columns:

```ts
import { createColumnHelper, cellRenderers } from '@meldui/vue'

const helper = createColumnHelper<MyData>()
```

### Badge Renderer

Renders the cell value as a styled Badge component with variant mapping:

```ts
helper.accessor('status', {
  title: 'Status',
  cell: cellRenderers.badge({
    variantMap: {
      active: 'default',
      inactive: 'secondary',
      pending: 'outline',
      error: 'destructive',
    },
    defaultVariant: 'secondary',
    format: (value) => value.toUpperCase(),
  }),
})
```

| Option           | Type                           | Default       | Description                                   |
| ---------------- | ------------------------------ | ------------- | --------------------------------------------- |
| `variantMap`     | `Record<string, BadgeVariant>` | `{}`          | Maps cell values to badge variants            |
| `defaultVariant` | `BadgeVariant`                 | `'secondary'` | Fallback variant when value is not in the map |
| `format`         | `(value: string) => string`    | capitalize    | Custom text formatter                         |

### Date Renderer

Formats date values using `Intl.DateTimeFormat`:

```ts
helper.accessor('createdAt', {
  title: 'Created',
  cell: cellRenderers.date({
    format: 'medium',
    locale: 'en-US',
    fallback: 'N/A',
  }),
})
```

| Option     | Type                                                                    | Default    | Description                    |
| ---------- | ----------------------------------------------------------------------- | ---------- | ------------------------------ |
| `format`   | `'short' \| 'medium' \| 'long' \| 'full' \| Intl.DateTimeFormatOptions` | `'medium'` | Date format                    |
| `locale`   | `string`                                                                | `'en-US'`  | Locale string                  |
| `fallback` | `string`                                                                | `'-'`      | Text for null/undefined values |

### Currency Renderer

Formats numbers as currency:

```ts
helper.accessor('price', {
  title: 'Price',
  cell: cellRenderers.currency({
    currency: 'EUR',
    locale: 'de-DE',
    minimumFractionDigits: 2,
  }),
})
```

| Option                  | Type     | Default   | Description                    |
| ----------------------- | -------- | --------- | ------------------------------ |
| `currency`              | `string` | `'USD'`   | ISO 4217 currency code         |
| `locale`                | `string` | `'en-US'` | Locale string                  |
| `minimumFractionDigits` | `number` | `0`       | Minimum decimal places         |
| `maximumFractionDigits` | `number` | `2`       | Maximum decimal places         |
| `fallback`              | `string` | `'-'`     | Text for null/undefined values |

### Number Renderer

Formats numbers with locale-aware formatting:

```ts
helper.accessor('quantity', {
  title: 'Quantity',
  cell: cellRenderers.number({
    useGrouping: true,
    compact: false,
  }),
})

// Compact notation
helper.accessor('views', {
  title: 'Views',
  cell: cellRenderers.number({ compact: true }), // e.g., "1.2K", "3.4M"
})
```

| Option                  | Type      | Default     | Description                    |
| ----------------------- | --------- | ----------- | ------------------------------ |
| `locale`                | `string`  | `'en-US'`   | Locale string                  |
| `minimumFractionDigits` | `number`  | `undefined` | Minimum decimal places         |
| `maximumFractionDigits` | `number`  | `undefined` | Maximum decimal places         |
| `useGrouping`           | `boolean` | `true`      | Use thousands separators       |
| `compact`               | `boolean` | `false`     | Compact notation (K, M, B)     |
| `fallback`              | `string`  | `'-'`       | Text for null/undefined values |

### Boolean Renderer

Renders boolean values as text or badges:

```ts
helper.accessor('isVerified', {
  title: 'Verified',
  cell: cellRenderers.boolean({
    trueText: 'Verified',
    falseText: 'Unverified',
    asBadge: true,
    trueVariant: 'default',
    falseVariant: 'secondary',
  }),
})
```

| Option         | Type           | Default       | Description                 |
| -------------- | -------------- | ------------- | --------------------------- |
| `trueText`     | `string`       | `'Yes'`       | Text for true values        |
| `falseText`    | `string`       | `'No'`        | Text for false values       |
| `asBadge`      | `boolean`      | `false`       | Render as a Badge component |
| `trueVariant`  | `BadgeVariant` | `'default'`   | Badge variant for true      |
| `falseVariant` | `BadgeVariant` | `'secondary'` | Badge variant for false     |

### Truncate Renderer

Truncates long text with an ellipsis and optional tooltip:

```ts
helper.accessor('description', {
  title: 'Description',
  cell: cellRenderers.truncate(50, { showTooltip: true }),
})
```

| Argument              | Type      | Default  | Description                                |
| --------------------- | --------- | -------- | ------------------------------------------ |
| `maxLength`           | `number`  | required | Maximum character length before truncation |
| `options.showTooltip` | `boolean` | `true`   | Show full text in a title tooltip on hover |

## Custom Cell Render Functions

For full control, provide a render function that returns a VNode or string:

```ts
import { h } from 'vue'

helper.accessor('email', {
  title: 'Email',
  cell: ({ getValue }) => {
    const email = getValue()
    return h('a', { href: `mailto:${email}`, class: 'text-primary underline' }, email)
  },
})

helper.accessor('progress', {
  title: 'Progress',
  cell: ({ getValue }) => {
    const value = getValue()
    return h('div', { class: 'flex items-center gap-2' }, [
      h('div', { class: 'h-2 w-20 bg-muted rounded-full overflow-hidden' }, [
        h('div', {
          class: 'h-full bg-primary rounded-full',
          style: { width: `${value}%` },
        }),
      ]),
      h('span', { class: 'text-sm text-muted-foreground' }, `${value}%`),
    ])
  },
})
```

The `cell` function receives a `CellContext` object with these useful properties:

| Property     | Description                                                             |
| ------------ | ----------------------------------------------------------------------- |
| `getValue()` | Returns the cell's value (typed based on the accessor)                  |
| `row`        | The TanStack Row object; access the full data object via `row.original` |
| `column`     | The TanStack Column instance                                            |
| `table`      | The TanStack Table instance                                             |

## Cell Slots

For cells that need Vue template syntax (instead of render functions), use named cell slots:

```vue
<DataTable :columns="columns" :data="data" ...>
  <template #cell-status="{ value, row }">
    <div class="flex items-center gap-2">
      <span
        class="h-2 w-2 rounded-full"
        :class="{
          'bg-green-500': value === 'active',
          'bg-red-500': value === 'inactive',
        }"
      />
      {{ value }}
    </div>
  </template>

  <template #cell-name="{ value, row }">
    <RouterLink :to="`/users/${row.original.id}`" class="text-primary hover:underline">
      {{ value }}
    </RouterLink>
  </template>
</DataTable>
```

Cell slot names follow the pattern `#cell-{columnId}` where `columnId` is the accessor key or column ID.

Each cell slot receives:

| Prop    | Type          | Description                                           |
| ------- | ------------- | ----------------------------------------------------- |
| `value` | `unknown`     | The cell's raw value                                  |
| `cell`  | `Cell<TData>` | TanStack Cell instance                                |
| `row`   | `Row<TData>`  | TanStack Row instance; access data via `row.original` |

## Footer Aggregations

Column footers can display aggregated values using the `aggregations` helpers:

```ts
import { aggregations } from '@meldui/vue'

helper.accessor('amount', {
  title: 'Amount',
  footer: aggregations.sum({ format: 'currency', currency: 'USD' }),
})

helper.accessor('id', {
  title: 'ID',
  footer: aggregations.count({ format: '{count} records' }),
})

helper.accessor('score', {
  title: 'Score',
  footer: aggregations.average({ decimals: 1 }),
})

helper.accessor('price', {
  title: 'Price',
  footer: aggregations.range({ format: 'currency' }),
})
```

### Available Aggregation Functions

| Function                        | Description           | Key Options                                            |
| ------------------------------- | --------------------- | ------------------------------------------------------ |
| `aggregations.sum(options)`     | Sum of all values     | `format`, `currency`, `decimals`, `prefix`, `suffix`   |
| `aggregations.count(options)`   | Row count             | `format` (e.g., `'{count} items'`), `locale`           |
| `aggregations.average(options)` | Average of all values | `format`, `decimals`, `currency`                       |
| `aggregations.min(options)`     | Minimum value         | `format` (including `'date'`), `dateOptions`, `prefix` |
| `aggregations.max(options)`     | Maximum value         | `format` (including `'date'`), `dateOptions`, `prefix` |
| `aggregations.range(options)`   | "min -- max" display  | `format`, `currency`, `prefix`, `suffix`               |
| `aggregations.custom(fn)`       | Custom computation    | Receives `{ table, column }` context                   |

### Custom Aggregation

```ts
helper.accessor('status', {
  title: 'Status',
  footer: aggregations.custom(({ table }) => {
    const rows = table.getFilteredRowModel().rows
    const completed = rows.filter((r) => r.original.status === 'completed').length
    return `${completed}/${rows.length} completed`
  }),
})
```

The footer section appears automatically when any column has a `footer` defined. It is sticky at the bottom of the scrollable table area.

## Row Styling

Customize row appearance dynamically based on row data:

```vue
<DataTable
  :columns="columns"
  :data="data"
  :page-count="pageCount"
  :on-server-side-change="handleServerSideChange"
  :row-class="
    (row) => ({
      'bg-red-50': row.original.status === 'error',
      'bg-yellow-50': row.original.status === 'warning',
      'opacity-50': row.original.isArchived,
    })
  "
  :row-style="
    (row) => ({
      borderLeft: row.original.priority === 'high' ? '3px solid red' : undefined,
    })
  "
  :row-props="
    (row) => ({
      'data-id': row.original.id,
      onClick: () => handleRowClick(row.original),
    })
  "
/>
```

| Prop       | Type                                                                  | Description                                            |
| ---------- | --------------------------------------------------------------------- | ------------------------------------------------------ |
| `rowClass` | `(row: Row<TData>) => string \| Record<string, boolean> \| undefined` | Dynamic CSS classes for each row                       |
| `rowStyle` | `(row: Row<TData>) => CSSProperties \| undefined`                     | Dynamic inline styles for each row                     |
| `rowProps` | `(row: Row<TData>) => HTMLAttributes \| undefined`                    | Additional HTML attributes/event handlers for each row |

## Header Styling

Set a CSS class on the table header row:

```vue
<DataTable :columns="columns" :data="data" header-class="bg-slate-100 dark:bg-slate-800" ... />
```

The default header class is `bg-muted`.

## Density

Control the vertical padding of table cells:

```vue
<DataTable :density="'compact'" ... />
```

| Value           | Description                               |
| --------------- | ----------------------------------------- |
| `'compact'`     | Minimal padding; fits more rows on screen |
| `'comfortable'` | Default balanced padding                  |
| `'spacious'`    | Extra padding for easier scanning         |

## Bordered Cells

Enable cell borders for a spreadsheet-like appearance:

```vue
<DataTable :bordered="true" ... />
```

## Column Resizing

Allow users to drag column borders to resize columns:

```vue
<DataTable
  :columns="columns"
  :data="data"
  :enable-column-resizing="true"
  column-resize-mode="onChange"
  ...
/>
```

| Prop                   | Type                    | Default      | Description                                         |
| ---------------------- | ----------------------- | ------------ | --------------------------------------------------- |
| `enableColumnResizing` | `boolean`               | `false`      | Enable drag-to-resize column handles                |
| `columnResizeMode`     | `'onChange' \| 'onEnd'` | `'onChange'` | When to apply size changes: live or after drag ends |

When column resizing is enabled, a drag handle appears on the right edge of each header cell. The table layout switches to `table-layout: fixed` for consistent resizing.

Control per-column resize behavior using the `size`, `minSize`, and `maxSize` options in column definitions.

## Column Hiding

Enable the column visibility toggle:

```vue
<DataTable :columns="columns" :data="data" :enable-column-hiding="true" ... />
```

A "View" button appears in the toolbar, opening a dropdown with toggles for each hideable column. Columns with `enableHiding: false` (like selection and actions) are not shown in this list.

## Template Slots

The DataTable provides multiple named slots for deep customization:

### Toolbar Slots

```vue
<DataTable ...>
  <!-- Completely replace the toolbar -->
  <template #toolbar="{ table, loading }">
    <MyCustomToolbar :table="table" :loading="loading" />
  </template>

  <!-- Or add content before/after default toolbar items -->
  <template #toolbar-start="{ table }">
    <MyCustomButton />
  </template>

  <template #toolbar-end="{ table }">
    <ExportButton :table="table" />
  </template>
</DataTable>
```

### Row and Cell Slots

```vue
<DataTable ...>
  <!-- Completely replace row rendering -->
  <template #row="{ row, cells, isSelected, index }">
    <tr :class="{ 'bg-blue-50': isSelected }">
      <td v-for="cell in cells" :key="cell.id">
        <!-- custom cell rendering -->
      </td>
    </tr>
  </template>

  <!-- Override specific column cells -->
  <template #cell-name="{ value, row }">
    <strong>{{ value }}</strong>
  </template>
</DataTable>
```

### State Slots

```vue
<DataTable ...>
  <!-- Custom empty state -->
  <template #empty="{ message }">
    <div class="flex flex-col items-center py-12">
      <IconInbox class="h-12 w-12 text-muted-foreground" />
      <p class="mt-2 text-muted-foreground">{{ message }}</p>
    </div>
  </template>

  <!-- Custom error state -->
  <template #error="{ error, retry }">
    <div class="p-4 text-center">
      <p class="text-destructive">{{ error }}</p>
      <button @click="retry">Try Again</button>
    </div>
  </template>
</DataTable>
```

### Expanded Row Slot

```vue
<DataTable :enable-row-expansion="true" ...>
  <template #expanded-row="{ row }">
    <div class="p-4 grid grid-cols-2 gap-4">
      <div>
        <h4 class="font-medium">Details</h4>
        <p>{{ row.original.description }}</p>
      </div>
      <div>
        <h4 class="font-medium">Metadata</h4>
        <pre>{{ JSON.stringify(row.original.meta, null, 2) }}</pre>
      </div>
    </div>
  </template>
</DataTable>
```

### Pagination Slot

```vue
<DataTable ...>
  <template #pagination="{ table, pageCount, currentPage, canPrevious, canNext }">
    <div class="flex justify-center gap-2">
      <button :disabled="!canPrevious" @click="table.previousPage()">Prev</button>
      <span>Page {{ currentPage }} of {{ pageCount }}</span>
      <button :disabled="!canNext" @click="table.nextPage()">Next</button>
    </div>
  </template>
</DataTable>
```

### Footer Slot

```vue
<DataTable ...>
  <template #footer="{ table, footerGroups }">
    <tfoot>
      <tr>
        <td :colspan="columns.length" class="text-center font-bold">
          Custom Footer Content
        </td>
      </tr>
    </tfoot>
  </template>
</DataTable>
```

## Slot Summary

| Slot              | Props                                                               | Description                                |
| ----------------- | ------------------------------------------------------------------- | ------------------------------------------ |
| `toolbar`         | `{ table, loading }`                                                | Replace entire toolbar                     |
| `toolbar-start`   | `{ table }`                                                         | Insert content at the start of the toolbar |
| `toolbar-end`     | `{ table }`                                                         | Insert content at the end of the toolbar   |
| `row`             | `{ row, cells, isSelected, index }`                                 | Replace entire row rendering               |
| `cell-{columnId}` | `{ value, cell, row }`                                              | Override a specific column's cell          |
| `empty`           | `{ message, columns }`                                              | Custom empty state                         |
| `error`           | `{ error, retry }`                                                  | Custom error state                         |
| `expanded-row`    | `{ row }`                                                           | Content for expanded rows                  |
| `pagination`      | `{ table, pageCount, currentPage, pageSize, canPrevious, canNext }` | Replace pagination                         |
| `footer`          | `{ table, footerGroups }`                                           | Replace table footer                       |
