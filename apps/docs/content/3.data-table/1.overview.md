# DataTable

The DataTable is a high-level composite component built on top of [TanStack Table](https://tanstack.com/table/latest) (`@tanstack/vue-table`). It provides a fully featured, server-side data grid with sorting, filtering, pagination, row selection, column pinning, bulk actions, keyboard navigation, and more.

## When to Use DataTable

Use the DataTable when you need to display tabular data with any combination of the following:

- **Server-side pagination** with configurable page sizes
- **Sortable columns** with ascending/descending toggle
- **Filtering** via simple or advanced (operator-based) filter modes
- **Row selection** with checkboxes and bulk actions
- **Column pinning** to freeze columns on the left or right edge
- **Column resizing** for user-adjustable column widths
- **Row expansion** for showing detail panels inline
- **Keyboard navigation** for accessible row traversal
- **Loading, error, and empty states** with built-in UI
- **Custom cell rendering** with pre-built renderers for badges, dates, currency, numbers, and booleans
- **Footer aggregations** (sum, count, average, min, max, range)

For simpler use cases where you only need a static table without interactivity, consider using the base `Table` component directly.

## Architecture

The DataTable is composed of several internal components that work together:

| Component                | Purpose                                                                            |
| ------------------------ | ---------------------------------------------------------------------------------- |
| `DataTable`              | Main wrapper component; orchestrates all sub-components                            |
| `DataTableToolbar`       | Search input, filter chips, bulk actions, refresh button, column visibility toggle |
| `DataTablePagination`    | Page navigation, page size selector, page info display                             |
| `DataTableColumnHeader`  | Column header with sort toggle, pin/unpin, and hide options in a dropdown          |
| `DataTableBulkActions`   | Dropdown menu for actions on selected rows                                         |
| `DataTableFilterCommand` | Command palette for adding filter instances                                        |
| `DataTableViewOptions`   | Column visibility toggle dropdown                                                  |
| `ActionsCellInline`      | Inline action buttons rendered in a row                                            |
| `ActionsCellDropdown`    | Dropdown menu for row-level actions                                                |

### Composables

| Composable         | Purpose                                                                                 |
| ------------------ | --------------------------------------------------------------------------------------- |
| `useDataTable`     | Core composable that creates the TanStack Table instance and manages all reactive state |
| `usePinnedColumns` | Manages CSS custom properties for pinned column offsets and scroll shadow feedback      |
| `useTableKeyboard` | Adds keyboard navigation (arrow keys, page up/down, selection, activation)              |

### Utilities

| Utility                     | Purpose                                                                                  |
| --------------------------- | ---------------------------------------------------------------------------------------- |
| `createColumnHelper`        | Type-safe factory for defining columns (accessor, display, selection, actions, expander) |
| `cellRenderers`             | Pre-built cell render functions (badge, date, currency, number, boolean, truncate)       |
| `aggregations`              | Footer aggregation helpers (sum, count, average, min, max, range, custom)                |
| `tableStateToServerParams`  | Converts TanStack table state to a server-friendly request format                        |
| `serverResponseToTableData` | Converts a paginated server response to the format DataTable expects                     |
| `defineFilter`              | Factory for registering custom filter plugins                                            |

## Server-Side by Design

The DataTable is designed exclusively for **server-side data operations**. All sorting, filtering, and pagination are manual -- meaning the component does not perform these operations on the client. Instead, it emits state changes through the `onServerSideChange` callback, and your application is responsible for fetching data from the server and passing it back.

```ts
// The DataTable calls this callback whenever sorting, filters, or pagination change
function handleServerSideChange({ sorting, filters, pagination }) {
  const params = tableStateToServerParams(
    { sorting, filters, pagination },
    filterFields,
    'name', // search column
  )
  // params is now a server-friendly object you can send to your API
  fetchData(params)
}
```

## Quick Example

Here is a minimal setup to give you a sense of how the pieces fit together. Each feature is covered in depth in the subsequent pages.

```vue
<script setup lang="ts">
import { ref } from 'vue'
import {
  DataTable,
  createColumnHelper,
  tableStateToServerParams,
  serverResponseToTableData,
} from '@meldui/vue'

interface User {
  id: number
  name: string
  email: string
}

const helper = createColumnHelper<User>()

const columns = [
  helper.accessor('name', { title: 'Name' }),
  helper.accessor('email', { title: 'Email' }),
]

const data = ref<User[]>([])
const pageCount = ref(0)

async function handleServerSideChange({ sorting, filters, pagination }) {
  const params = tableStateToServerParams({ sorting, filters, pagination })
  const response = await fetch(`/api/users?${new URLSearchParams(params)}`)
  const json = await response.json()
  const result = serverResponseToTableData(json)
  data.value = result.data
  pageCount.value = result.pageCount
}
</script>

<template>
  <DataTable
    :columns="columns"
    :data="data"
    :page-count="pageCount"
    :on-server-side-change="handleServerSideChange"
  />
</template>
```
