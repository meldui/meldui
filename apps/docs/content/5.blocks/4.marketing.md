Marketing blocks are page-level patterns for public-facing content such as landing pages, pricing pages, and feature showcases. They rely on Card, Button, Badge, and Separator alongside standard HTML elements styled with Tailwind CSS v4 utilities.

## Hero Section

A hero section introduces your product with a headline, description, and call-to-action buttons. Center the content and use a maximum width to keep line lengths readable.

```vue
<script setup lang="ts">
import { IconArrowRight } from '@meldui/tabler-vue'
import { Badge, Button } from '@meldui/vue'
</script>

<template>
  <section class="flex flex-col items-center text-center px-4 py-24 md:py-32">
    <Badge variant="secondary" class="mb-4">Now in Beta</Badge>

    <h1 class="text-4xl font-bold tracking-tight sm:text-5xl md:text-6xl max-w-3xl">
      Build better products with
      <span class="text-primary">MeldUI</span>
    </h1>

    <p class="mt-6 text-lg text-muted-foreground max-w-2xl">
      A comprehensive design system with 60+ components, built on shadcn-vue. Ship polished
      interfaces faster with consistent, accessible building blocks.

    <div class="mt-10 flex flex-wrap items-center justify-center gap-4">

        Get Started

      </Button>
      <Button size="lg" variant="outline">View Documentation</Button>
    </div>
  </section>

```

## Design Guidance

**Visual hierarchy.** Hero sections use the largest text sizes (`text-4xl` to `text-6xl`) and generous vertical padding. Feature grids and pricing use `text-3xl` headings. Keep body text at `text-lg` for readability.

**Highlighting.** Draw attention to the recommended pricing tier with `border-primary ring-1 ring-primary` and a `Badge`. Only highlight one option to avoid decision paralysis.

**Spacing.** Use `max-w-5xl mx-auto` to constrain content width on wide screens. Marketing pages benefit from more generous padding (`py-16 md:py-24`) than application pages.

**Responsiveness.** All grids start at a single column on mobile and expand: `sm:grid-cols-2` for features, `md:grid-cols-3` for pricing and testimonials. Test the layout at every breakpoint.
