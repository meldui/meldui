Authentication blocks are page-level patterns for user sign-in flows. They combine Card, Input, Button, Label, and Form components into polished, ready-to-use layouts. Each pattern centers the form on the page and uses vee-validate with Zod for schema-driven validation.

## Login Form

A standard email and password login card with a link to the forgot-password flow. The form validates both fields before submission.

```vue
<script setup lang="ts">
import { IconLock, IconMail } from '@meldui/tabler-vue'
import {
  Button,
  Card,
  CardContent,
  CardDescription,
  CardFooter,
  CardHeader,
  CardTitle,
  Checkbox,
  FormControl,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
  Input,
  Separator,
} from '@meldui/vue'
import { toTypedSchema } from '@vee-validate/zod'
import { useForm } from 'vee-validate'
import { z } from 'zod'

const loginSchema = z.object({
  email: z.string().email('Please enter a valid email address'),
  password: z.string().min(8, 'Password must be at least 8 characters'),
  rememberMe: z.boolean().default(false),
})

const { handleSubmit } = useForm({
  validationSchema: toTypedSchema(loginSchema),
  initialValues: {
    email: '',
    password: '',
    rememberMe: false,
  },
})

const onSubmit = handleSubmit((values) => {
  // Send values to your authentication API
  console.log('Login:', values)
})
</script>

<template>
  <div class="flex min-h-screen items-center justify-center bg-muted/40 px-4">
    <Card class="w-full max-w-md">
      <CardHeader class="text-center">
        <CardTitle class="text-2xl">Welcome back</CardTitle>
        <CardDescription>Sign in to your account to continue</CardDescription>
      </CardHeader>
      <CardContent>
        <form class="space-y-4" @submit="onSubmit">
          <FormField v-slot="{ componentField }" name="email">
            <FormItem>
              <FormLabel>Email</FormLabel>
              <FormControl>
                <Input type="email" placeholder="you@example.com" v-bind="componentField" />
              </FormControl>
              <FormMessage />
            </FormItem>
          </FormField>

          <FormField v-slot="{ componentField }" name="password">
            <FormItem>
              <div class="flex items-center justify-between">
                <FormLabel>Password</FormLabel>
                <a href="/forgot-password" class="text-sm text-primary hover:underline">
                  Forgot password?
                </a>
              </div>
              <FormControl>
                <Input type="password" placeholder="Enter your password" v-bind="componentField" />
              </FormControl>
              <FormMessage />
            </FormItem>
          </FormField>

          <FormField v-slot="{ value, handleChange }" name="rememberMe">
            <FormItem class="flex items-center gap-2">
              <FormControl>
                <Checkbox :model-value="value" @update:model-value="handleChange" />
              </FormControl>
              <FormLabel class="!mt-0 text-sm font-normal">Remember me</FormLabel>
            </FormItem>
          </FormField>

          <Button type="submit" class="w-full">Sign in</Button>
        </form>
      </CardContent>
      <Separator />
      <CardFooter class="justify-center pt-6">
        <p class="text-sm text-muted-foreground">
          Don't have an account?
          <a href="/register" class="text-primary hover:underline">Create one</a>
        </p>
      </CardFooter>
    </Card>
  </div>
</template>
```

## Registration Form

A registration card that collects name, email, and password with confirmation. The Zod schema uses `.refine()` to ensure the passwords match.

```vue
<script setup lang="ts">
import {
  Button,
  Card,
  CardContent,
  CardDescription,
  CardFooter,
  CardHeader,
  CardTitle,
  FormControl,
  FormDescription,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
  Input,
  Separator,
} from '@meldui/vue'
import { toTypedSchema } from '@vee-validate/zod'
import { useForm } from 'vee-validate'
import { z } from 'zod'

const registerSchema = z
  .object({
    name: z.string().min(2, 'Name must be at least 2 characters'),
    email: z.string().email('Please enter a valid email address'),
    password: z
      .string()
      .min(8, 'Password must be at least 8 characters')
      .regex(/[A-Z]/, 'Password must contain an uppercase letter')
      .regex(/[0-9]/, 'Password must contain a number'),
    confirmPassword: z.string(),
  })
  .refine((data) => data.password === data.confirmPassword, {
    message: 'Passwords do not match',
    path: ['confirmPassword'],
  })

const { handleSubmit } = useForm({
  validationSchema: toTypedSchema(registerSchema),
  initialValues: {
    name: '',
    email: '',
    password: '',
    confirmPassword: '',
  },
})

const onSubmit = handleSubmit((values) => {
  console.log('Register:', values)
})
</script>

<template>
  <div class="flex min-h-screen items-center justify-center bg-muted/40 px-4">
    <Card class="w-full max-w-md">
      <CardHeader class="text-center">
        <CardTitle class="text-2xl">Create an account</CardTitle>
        <CardDescription>Enter your details to get started</CardDescription>
      </CardHeader>
      <CardContent>
        <form class="space-y-4" @submit="onSubmit">
          <FormField v-slot="{ componentField }" name="name">
            <FormItem>
              <FormLabel>Full Name</FormLabel>
              <FormControl>
                <Input placeholder="Jane Doe" v-bind="componentField" />
              </FormControl>
              <FormMessage />
            </FormItem>
          </FormField>

          <FormField v-slot="{ componentField }" name="email">
            <FormItem>
              <FormLabel>Email</FormLabel>
              <FormControl>
                <Input type="email" placeholder="you@example.com" v-bind="componentField" />
              </FormControl>
              <FormMessage />
            </FormItem>
          </FormField>

          <FormField v-slot="{ componentField }" name="password">
            <FormItem>
              <FormLabel>Password</FormLabel>
              <FormControl>
                <Input type="password" placeholder="Create a password" v-bind="componentField" />
              </FormControl>
              <FormDescription>
                At least 8 characters with one uppercase letter and one number.
              </FormDescription>
              <FormMessage />
            </FormItem>
          </FormField>

          <FormField v-slot="{ componentField }" name="confirmPassword">
            <FormItem>
              <FormLabel>Confirm Password</FormLabel>
              <FormControl>
                <Input type="password" placeholder="Repeat your password" v-bind="componentField" />
              </FormControl>
              <FormMessage />
            </FormItem>
          </FormField>

          <Button type="submit" class="w-full">Create account</Button>
        </form>
      </CardContent>
      <Separator />
      <CardFooter class="justify-center pt-6">
        <p class="text-sm text-muted-foreground">
          Already have an account?
          <a href="/login" class="text-primary hover:underline">Sign in</a>
        </p>
      </CardFooter>
    </Card>
  </div>
</template>
```

## Forgot Password

A minimal card that asks for the user's email and sends a password-reset link. After submission you can swap the form content to a confirmation message.

```vue
<script setup lang="ts">
import { IconArrowLeft } from '@meldui/tabler-vue'
import {
  Button,
  Card,
  CardContent,
  CardDescription,
  CardFooter,
  CardHeader,
  CardTitle,
  FormControl,
  FormField,
  FormItem,
  FormLabel,
  FormMessage,
  Input,
} from '@meldui/vue'
import { toTypedSchema } from '@vee-validate/zod'
import { useForm } from 'vee-validate'
import { ref } from 'vue'
import { z } from 'zod'

const submitted = ref(false)

const schema = z.object({
  email: z.string().email('Please enter a valid email address'),
})

const { handleSubmit } = useForm({
  validationSchema: toTypedSchema(schema),
  initialValues: { email: '' },
})

const onSubmit = handleSubmit((values) => {
  // Call your password-reset API
  console.log('Reset password for:', values.email)
  submitted.value = true
})
</script>

<template>
  <div class="flex min-h-screen items-center justify-center bg-muted/40 px-4">
    <Card class="w-full max-w-md">
      <CardHeader class="text-center">
        <CardTitle class="text-2xl">Reset your password</CardTitle>
        <CardDescription>
          Enter your email and we'll send you a link to reset your password.
        </CardDescription>
      </CardHeader>
      <CardContent>
        <!-- Success state -->
        <div v-if="submitted" class="text-center space-y-4">
          <p class="text-sm text-muted-foreground">
            If an account exists for that email, you'll receive a reset link shortly.
          </p>
          <Button variant="outline" class="w-full" @click="submitted = false">
            Try another email
          </Button>
        </div>

        <!-- Form state -->
        <form v-else class="space-y-4" @submit="onSubmit">
          <FormField v-slot="{ componentField }" name="email">
            <FormItem>
              <FormLabel>Email</FormLabel>
              <FormControl>
                <Input type="email" placeholder="you@example.com" v-bind="componentField" />
              </FormControl>
              <FormMessage />
            </FormItem>
          </FormField>

          <Button type="submit" class="w-full">Send reset link</Button>
        </form>
      </CardContent>
      <CardFooter class="justify-center">
        <a
          href="/login"
          class="inline-flex items-center gap-1 text-sm text-primary hover:underline"
        >
          <IconArrowLeft :size="16" />
          Back to login
        </a>
      </CardFooter>
    </Card>
  </div>
</template>
```

## Design Guidance

**Layout.** Center authentication cards vertically and horizontally on the page using `flex min-h-screen items-center justify-center`. A `max-w-md` card width works well on both mobile and desktop.

**Validation.** Use Zod schemas with vee-validate for consistent, declarative validation. Display inline errors with `FormMessage` and provide helper text via `FormDescription` where rules are non-obvious (e.g., password strength requirements).

**Navigation.** Always provide a way to move between login, registration, and forgot-password pages. Place these links in the `CardFooter` below a `Separator`.

**Accessibility.** Every input must be wrapped with `FormField`, `FormItem`, `FormLabel`, and `FormControl` so that labels are properly associated with their inputs and screen readers announce validation errors.
