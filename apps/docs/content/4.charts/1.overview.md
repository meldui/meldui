## What is @meldui/charts-vue?

`@meldui/charts-vue` is a **separate package** from `@meldui/vue` that provides data visualization components for Vue 3 applications. It is built on [Apache ECharts](https://echarts.apache.org/) with full tree-shaking support, so only the chart types you use are included in your final bundle.

The chart library provides a declarative, type-safe API that abstracts away ECharts internals. You describe your chart using a simple configuration object, and the component handles all rendering, theming, resizing, and event normalization.

## Architecture

The chart system is split across two packages:

| Package              | Purpose                                                                                                                |
| -------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `@meldui/charts-vue` | Chart components (`MeldLineChart`, `MeldBarChart`, etc.) and their configuration types                                 |
| `@meldui/vue`        | Optional utilities: `ChartContainer`, `ChartLegendContent`, `ChartTooltipContent`, `ChartCrosshair` (from @unovis/vue) |

`@meldui/charts-vue` has `@meldui/vue` as a peer dependency since it uses shared primitives like `Skeleton` for loading states.

## Available Chart Types

| Component          | Description                              | Best For                           |
| ------------------ | ---------------------------------------- | ---------------------------------- |
| `MeldLineChart`    | Line chart with optional data points     | Trends over time                   |
| `MeldAreaChart`    | Filled area chart                        | Cumulative values, comparisons     |
| `MeldBarChart`     | Vertical or horizontal bars              | Category comparisons               |
| `MeldPieChart`     | Pie chart                                | Proportions of a whole             |
| `MeldDonutChart`   | Pie chart with a hollow center           | Proportions with a summary value   |
| `MeldScatterChart` | Scatter plot                             | Correlations between two variables |
| `MeldRadarChart`   | Spider/radar chart                       | Multi-variable comparison          |
| `MeldHeatmapChart` | Color-intensity grid                     | Matrix data, distributions         |
| `MeldMixedChart`   | Combined line, bar, and area             | Multi-metric dashboards            |
| `MeldChart`        | Dynamic wrapper that lazy-loads any type | Config-driven rendering            |

## Key Features

- **Type-safe configuration** -- each chart type has its own TypeScript config interface (e.g., `MeldLineChartConfig`, `MeldBarChartConfig`) with only the properties relevant to that chart type.
- **Automatic theming** -- charts read your Tailwind CSS v4 design tokens and respond to dark mode changes in real time.
- **Built-in color palettes** -- 10 named palettes (`default`, `vibrant`, `pastel`, `ocean`, `accessible`, and more) with automatic dark-mode adjustments.
- **Responsive by default** -- every chart uses `ResizeObserver` to resize automatically when its container changes size.
- **Loading skeletons** -- a `loading` prop on every chart swaps the canvas for a skeleton that matches the chart type.
- **Normalized events** -- click, hover, legend selection, zoom, and brush events are normalized into a consistent MeldUI format, hiding ECharts internals.
- **Accessibility** -- each chart renders with `role="img"` and an auto-generated or custom `ariaLabel`.
- **Advanced escape hatch** -- every config accepts an `advanced` property where you can pass raw ECharts options for edge cases.

## Shared Props

All chart components accept the following base props:

| Prop        | Type                       | Default        | Description                                   |
| ----------- | -------------------------- | -------------- | --------------------------------------------- |
| `config`    | Chart-specific config type | _required_     | The chart configuration object.               |
| `height`    | `number \| string`         | `350`          | Chart height in pixels or CSS value.          |
| `width`     | `number \| string`         | `'100%'`       | Chart width in pixels or CSS value.           |
| `title`     | `string`                   | `undefined`    | Title rendered above the chart.               |
| `loading`   | `boolean`                  | `false`        | Show a loading skeleton instead of the chart. |
| `ariaLabel` | `string`                   | Auto-generated | Accessible label for screen readers.          |

## Shared Events

All chart components emit these events:

| Event          | Payload Type             | Description                         |
| -------------- | ------------------------ | ----------------------------------- |
| `click`        | `ChartClickEvent`        | A data point or series was clicked. |
| `hover`        | `ChartHoverEvent`        | A data point is being hovered.      |
| `mouseout`     | `ChartMouseOutEvent`     | The mouse left a data point.        |
| `legendSelect` | `ChartLegendSelectEvent` | A legend item was toggled.          |
| `dataZoom`     | `ChartDataZoomEvent`     | The zoom range changed.             |
| `brushSelect`  | `ChartBrushSelectEvent`  | A brush selection was made.         |

## Slots

Every chart component provides two slots for custom content:

| Slot     | Description                                                              |
| -------- | ------------------------------------------------------------------------ |
| `header` | Content above the chart. Falls back to the `title` prop if not provided. |
| `footer` | Content below the chart (e.g., custom legends or captions).              |

## Quick Example

```vue
<script setup lang="ts">
import { MeldBarChart } from '@meldui/charts-vue'
import type { MeldBarChartConfig } from '@meldui/charts-vue'

const config: MeldBarChartConfig = {
  series: [
    { name: 'Revenue', data: [420, 380, 510, 590, 480, 620] },
    { name: 'Expenses', data: [320, 290, 350, 410, 370, 430] },
  ],
  xAxis: {
    categories: ['Jan', 'Feb', 'Mar', 'Apr', 'May', 'Jun'],
  },
  colors: 'ocean',
  tooltip: { shared: true },
}
</script>

<template>
  <MeldBarChart title="Revenue vs Expenses" :config="config" :height="400" />
</template>
```
