## How Theming Works

MeldUI Charts automatically integrate with your application's theme. The chart components use the `useChartTheme` composable internally to:

1. **Read CSS custom properties** from your Tailwind CSS v4 theme (e.g., `--color-primary`, `--color-success`).
2. **Detect dark mode** by watching for the `dark` class on `document.documentElement`.
3. **Adjust colors** for optimal contrast in both light and dark modes.

No configuration is needed for basic theming -- charts will inherit your application's design tokens automatically.

## Color Palettes

Every chart accepts a `colors` property that controls how series colors are assigned. There are three approaches:

### Auto (Default)

When `colors` is `'auto'` or omitted, chart colors are derived from your Tailwind CSS v4 theme tokens (`--color-primary`, `--color-success`, `--color-warning`, `--color-danger`, `--color-info`, etc.):

```vue
<script setup lang="ts">
import { MeldBarChart } from '@meldui/charts-vue'
import type { MeldBarChartConfig } from '@meldui/charts-vue'

const config: MeldBarChartConfig = {
  series: [
    { name: 'Series A', data: [10, 20, 30] },
    { name: 'Series B', data: [15, 25, 35] },
  ],
  xAxis: { categories: ['Q1', 'Q2', 'Q3'] },
  colors: 'auto', // This is the default
}
</script>

<template>
  <MeldBarChart :config="config" />
</template>
```

### Named Palette

Use one of the 10 built-in palettes by passing its name as a string:

```vue
<script setup lang="ts">
import { MeldLineChart } from '@meldui/charts-vue'
import type { MeldLineChartConfig } from '@meldui/charts-vue'

const config: MeldLineChartConfig = {
  series: [
    { name: 'Alpha', data: [10, 20, 30, 40, 50] },
    { name: 'Beta', data: [15, 25, 35, 45, 55] },
    { name: 'Gamma', data: [12, 22, 32, 42, 52] },
  ],
  xAxis: { categories: ['Jan', 'Feb', 'Mar', 'Apr', 'May'] },
  colors: 'ocean',
}
</script>

<template>
  <MeldLineChart title="Ocean Palette" :config="config" />
</template>
```

### Available Palettes

| Palette      | Description                                                       |
| ------------ | ----------------------------------------------------------------- |
| `default`    | Professional blues and purples. Ideal for business presentations. |
| `vibrant`    | Bold, saturated colors spanning the full spectrum.                |
| `pastel`     | Soft, muted colors for elegant presentations.                     |
| `monochrome` | Shades of blue-gray for minimal aesthetics.                       |
| `earth`      | Natural warm tones (browns, greens, oranges).                     |
| `ocean`      | Cool blues and teals inspired by water and sky.                   |
| `sunset`     | Warm reds, oranges, and purples.                                  |
| `corporate`  | Conservative business colors for professional reports.            |
| `neon`       | Bright, modern colors for tech applications.                      |
| `accessible` | High contrast, colorblind-safe palette for maximum accessibility. |

### Custom Color Array

Pass an explicit array of hex colors for full control:

```vue
<script setup lang="ts">
import { MeldPieChart } from '@meldui/charts-vue'
import type { MeldPieChartConfig } from '@meldui/charts-vue'

const config: MeldPieChartConfig = {
  series: [
    { name: 'Category A', data: [40] },
    { name: 'Category B', data: [30] },
    { name: 'Category C', data: [20] },
    { name: 'Category D', data: [10] },
  ],
  colors: ['#1e40af', '#3b82f6', '#93c5fd', '#dbeafe'],
}
</script>

<template>
  <MeldPieChart title="Custom Blue Palette" :config="config" />
</template>
```

## Per-Series Colors

Override colors on individual series using the `color` property on each series object. This takes priority over the `colors` config:

```vue
<script setup lang="ts">
import { MeldBarChart } from '@meldui/charts-vue'
import type { MeldBarChartConfig } from '@meldui/charts-vue'

const config: MeldBarChartConfig = {
  series: [
    { name: 'Success', data: [85, 72, 90], color: '#10b981' },
    { name: 'Warning', data: [12, 18, 8], color: '#f59e0b' },
    { name: 'Error', data: [3, 10, 2], color: '#ef4444' },
  ],
  xAxis: { categories: ['Jan', 'Feb', 'Mar'] },
}
</script>

<template>
  <MeldBarChart title="Status Breakdown" :config="config" />
</template>
```

## Generating Colors Programmatically

The `generateColors` function is exported from `@meldui/charts-vue` for generating palette colors in your own code:

```typescript
import { generateColors } from '@meldui/charts-vue'

// Generate 5 colors from the "vibrant" palette
const colors = generateColors('vibrant', 5)
// Returns: ['#e63946', '#f77f00', '#06d6a0', '#118ab2', '#073b4c']

// Generate dark-mode optimized colors
const darkColors = generateColors('vibrant', 5, true)
```

### Function Signature

```typescript
function generateColors(paletteName: string, count: number, isDarkMode?: boolean): string[]
```

| Parameter     | Type      | Description                                            |
| ------------- | --------- | ------------------------------------------------------ |
| `paletteName` | `string`  | Name of the palette (e.g., `'vibrant'`, `'ocean'`).    |
| `count`       | `number`  | Number of colors to generate.                          |
| `isDarkMode`  | `boolean` | Adjust colors for dark backgrounds (default: `false`). |

## Dark Mode

Charts respond to dark mode automatically. The `useChartTheme` composable watches for changes to the `dark` class on `<html>` using a `MutationObserver`. When dark mode is toggled:

- Tooltip backgrounds switch to `hsl(var(--popover))`.
- Text colors switch to `hsl(var(--popover-foreground))`.
- Axis labels use `hsl(var(--muted-foreground))`.
- Grid lines and borders use subtle opacity values.
- Named palette colors are automatically adjusted (saturation and lightness tuned for dark backgrounds).

No action is needed from your side as long as your dark mode implementation toggles the `dark` class on `document.documentElement`.

### Dark Mode Color Adjustments

Different palette types receive different adjustments:

| Palette Type                       | Adjustment                                                  |
| ---------------------------------- | ----------------------------------------------------------- |
| High saturation (vibrant, neon)    | Saturation reduced, lightness increased to avoid harshness. |
| Already light (pastel)             | Minimal adjustment since they are already soft.             |
| Low saturation (monochrome)        | Moderate lightness increase for contrast.                   |
| Standard (earth, ocean, corporate) | Balanced lightness increase.                                |

## Tooltip Theming

Chart tooltips use your Tailwind CSS v4 design tokens automatically:

- **Background:** `hsl(var(--popover))`
- **Border:** `hsl(var(--border))`
- **Text:** `hsl(var(--popover-foreground))`
- **Font:** Inherited from your CSS (`font-family: inherit`)

To customize tooltip behavior per chart, use the `tooltip` config property:

```vue
<script setup lang="ts">
import { MeldLineChart } from '@meldui/charts-vue'
import type { MeldLineChartConfig } from '@meldui/charts-vue'

const config: MeldLineChartConfig = {
  series: [{ name: 'Revenue', data: [4200, 3800, 5100, 5900] }],
  xAxis: { categories: ['Q1', 'Q2', 'Q3', 'Q4'] },
  tooltip: {
    enabled: true,
    shared: true,
    formatter: (value: number, seriesName: string) => {
      return `${seriesName}: $${value.toLocaleString()}`
    },
  },
}
</script>

<template>
  <MeldLineChart :config="config" />
</template>
```

## Axis and Grid Styling

Axes and grid lines are styled with subtle defaults that work in both light and dark modes:

- **Axis lines:** `rgba(128, 128, 128, 0.30)` -- subtle gray at 30% opacity.
- **Grid lines:** `rgba(128, 128, 128, 0.25)` -- dashed lines at 25% opacity.
- **Axis labels:** `hsl(var(--muted-foreground))` -- muted text color.

To customize grid visibility per chart:

```vue
<script setup lang="ts">
import { MeldLineChart } from '@meldui/charts-vue'
import type { MeldLineChartConfig } from '@meldui/charts-vue'

const config: MeldLineChartConfig = {
  series: [{ name: 'Data', data: [10, 20, 15, 30, 25] }],
  xAxis: {
    categories: ['A', 'B', 'C', 'D', 'E'],
    labels: { show: true, rotate: -45 },
  },
  yAxis: {
    title: 'Value',
    min: 0,
    max: 40,
  },
  grid: {
    show: true,
    strokeDashArray: 4,
  },
}
</script>

<template>
  <MeldLineChart :config="config" />
</template>
```

## Legend Customization

Control legend visibility, position, and alignment:

```vue
<script setup lang="ts">
import { MeldBarChart } from '@meldui/charts-vue'
import type { MeldBarChartConfig } from '@meldui/charts-vue'

const config: MeldBarChartConfig = {
  series: [
    { name: 'Online', data: [320, 410, 380] },
    { name: 'In-Store', data: [220, 280, 310] },
  ],
  xAxis: { categories: ['Jan', 'Feb', 'Mar'] },
  legend: {
    show: true,
    position: 'bottom',
    align: 'center',
  },
}
</script>

<template>
  <MeldBarChart :config="config" />
</template>
```

| Position   | Description                       |
| ---------- | --------------------------------- |
| `'top'`    | Legend above the chart (default). |
| `'bottom'` | Legend below the chart.           |
| `'left'`   | Legend to the left of the chart.  |
| `'right'`  | Legend to the right of the chart. |

## Animation Configuration

Charts animate by default with a 400ms `cubicOut` easing. Disable animations with `animations: false`:

```vue
<script setup lang="ts">
import { MeldBarChart } from '@meldui/charts-vue'
import type { MeldBarChartConfig } from '@meldui/charts-vue'

const config: MeldBarChartConfig = {
  series: [{ name: 'Data', data: [10, 20, 30, 40] }],
  xAxis: { categories: ['A', 'B', 'C', 'D'] },
  animations: false,
}
</script>

<template>
  <MeldBarChart :config="config" />
</template>
```

## Advanced Escape Hatch

For edge cases where the MeldUI abstraction is not sufficient, every chart config accepts an `advanced` property. This object is deep-merged with the generated ECharts options, giving you access to the full ECharts API:

```vue
<script setup lang="ts">
import { MeldLineChart } from '@meldui/charts-vue'
import type { MeldLineChartConfig } from '@meldui/charts-vue'

const config: MeldLineChartConfig = {
  series: [{ name: 'Values', data: [10, 20, 15, 30, 25] }],
  xAxis: { categories: ['A', 'B', 'C', 'D', 'E'] },
  advanced: {
    // Raw ECharts options -- use sparingly
    toolbox: {
      show: true,
      feature: {
        saveAsImage: { show: true },
        dataView: { show: true },
      },
    },
    // Add a mark line at y=20
    series: [
      {
        markLine: {
          data: [{ yAxis: 20 }],
          lineStyle: { color: '#ef4444', type: 'dashed' },
          label: { formatter: 'Threshold' },
        },
      },
    ],
  },
}
</script>

<template>
  <MeldLineChart title="With Advanced Options" :config="config" />
</template>
```

Use `advanced` only when the standard config properties do not cover your use case. The `advanced` object is typed as `Record<string, unknown>` to avoid exposing ECharts types in the public API.

## Maximum Recommended Series

The palette system works best with up to 12 series (`MAX_RECOMMENDED_SERIES`). Beyond this count, colors become difficult to distinguish visually. If you have more than 12 categories, consider grouping less significant items into an "Other" category or using a heatmap chart instead.

```typescript
import { MAX_RECOMMENDED_SERIES } from '@meldui/charts-vue'

console.log(MAX_RECOMMENDED_SERIES) // 12
```
