## Prerequisites

Before installing the chart package, make sure your project already has:

- **@meldui/vue** -- the core MeldUI component library (required peer dependency)
- **Vue 3.4+**
- **Tailwind CSS v4** -- configured with MeldUI theme tokens (see [Getting Started > Installation](/getting-started/installation))

## Step 1: Install the Package

`@meldui/charts-vue` is distributed on GitHub Package Registry alongside the other MeldUI packages. If you have already configured the `@meldui` scope in your `.npmrc`, you can install directly:

```bash
pnpm add @meldui/charts-vue
```

If you have not configured the registry yet, add this to your project `.npmrc` first:

```bash
@meldui:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${NPM_TOKEN}
```

## Step 2: Verify Peer Dependencies

The chart package declares two peer dependencies:

| Peer Dependency | Required Version   |
| --------------- | ------------------ |
| `vue`           | `^3.4.0`           |
| `@meldui/vue`   | workspace (latest) |

If you already have `@meldui/vue` and `vue` installed, no extra action is needed. Otherwise:

```bash
pnpm add @meldui/vue @meldui/tabler-vue vue
```

## Step 3: Confirm CSS Setup

Charts inherit their fonts, colors, and tooltip styles from your Tailwind CSS v4 theme tokens. Your CSS entry file should already include the MeldUI theme:

```css
@import 'tailwindcss';
@import 'tw-animate-css';
@import '@meldui/vue/themes/default';

@source "../**/*.{vue,js,ts,jsx,tsx}";
@source "../node_modules/@meldui/vue/dist/**/*.mjs";
```

No additional CSS is required for `@meldui/charts-vue`. The chart components render to a `<canvas>` element and read CSS custom properties at runtime for theming.

## Step 4: Use a Chart Component

```vue
<script setup lang="ts">
import { MeldLineChart } from '@meldui/charts-vue'
import type { MeldLineChartConfig } from '@meldui/charts-vue'

const config: MeldLineChartConfig = {
  series: [{ name: 'Users', data: [120, 200, 150, 280, 320, 410] }],
  xAxis: {
    categories: ['Jan', 'Feb', 'Mar', 'Apr', 'May', 'Jun'],
  },
}
</script>

<template>
  <MeldLineChart title="Monthly Active Users" :config="config" />
</template>
```

## What Gets Installed

`@meldui/charts-vue` bundles the following as regular dependencies (you do not need to install them yourself):

| Dependency         | Purpose                 |
| ------------------ | ----------------------- |
| `echarts` (v6)     | Core charting engine    |
| `vue-echarts` (v8) | Vue 3 integration layer |

Both are tree-shakeable. The chart package only registers the ECharts modules needed for the chart types you actually use.

## Bundle Size

Because the library uses ECharts modular imports (`echarts/core`), your bundle will only include the renderers and series types that are referenced. A single `MeldLineChart` adds approximately 150-200 KB gzipped to your bundle. Adding more chart types from the same package has minimal additional cost because they share the core ECharts renderer.

## TypeScript Support

All chart components and configuration types are fully typed. Import types directly from the package:

```typescript
import type {
  MeldLineChartConfig,
  MeldBarChartConfig,
  MeldAreaChartConfig,
  MeldPieChartConfig,
  MeldDonutChartConfig,
  MeldScatterChartConfig,
  MeldRadarChartConfig,
  MeldHeatmapChartConfig,
  MeldMixedChartConfig,
  ChartClickEvent,
  ChartHoverEvent,
} from '@meldui/charts-vue'
```

## Dynamic Chart Component

If you need to render a chart type determined at runtime (e.g., from user preferences or a config file), use the `MeldChart` wrapper. It uses `defineAsyncComponent` internally to lazy-load only the chart type that is actually rendered:

```vue
<script setup lang="ts">
import { MeldChart } from '@meldui/charts-vue'
import type { MeldBarChartConfig } from '@meldui/charts-vue'

const chartType = 'bar'
const chartConfig: MeldBarChartConfig = {
  series: [{ name: 'Sales', data: [10, 20, 30, 40] }],
  xAxis: { categories: ['Q1', 'Q2', 'Q3', 'Q4'] },
}
</script>

<template>
  <MeldChart :type="chartType" :config="chartConfig" />
</template>
```

## Troubleshooting

### Charts render with no colors

Make sure your CSS includes the MeldUI theme (`@import '@meldui/vue/themes/default'`). The chart components read CSS custom properties like `--color-primary` for their default palette.

### Charts do not resize when container changes

Every chart component automatically uses `ResizeObserver`. If charts are not resizing, verify that the chart's parent element has a defined width. A common mistake is placing a chart inside a container with no explicit width or `width: 0`.

### Dark mode does not update charts

The chart theme composable watches for the `dark` class on `document.documentElement`. Make sure your dark mode implementation toggles this class (the standard approach for Tailwind CSS v4).
