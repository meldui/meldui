`DocumentViewer` is built on EmbedPDF's plugin architecture. Each plugin handles one concern (zoom, search, annotations, etc.). All Phase 1 plugins are **always registered**; the `features` flags control toolbar / UX visibility, not plugin presence (see [Plugin registration pattern](#plugin-registration-pattern) below). Only the Phase 2 plugins are gated by their flag. This page lists every plugin in scope and links to its reference.

## Always-on (no feature flag)

These five plugins constitute the **minimum** PDF rendering pipeline. They're always registered:

- **[document-manager](/docs/document-viewer/plugins/document-manager)** ‚Äî orchestrates document lifecycle (load, switch, unload)
- **[viewport](/docs/document-viewer/plugins/viewport)** ‚Äî owns the scrollable container DOM
- **[scroll](/docs/document-viewer/plugins/scroll)** ‚Äî page-aware virtual scrolling
- **[render](/docs/document-viewer/plugins/render)** ‚Äî paints pages to canvas
- **[tiling](/docs/document-viewer/plugins/tiling)** ‚Äî splits large pages into tiles for memory efficiency

## View controls

Always registered. The listed `features` flag toggles the corresponding toolbar UI; the plugin (and its composables) stay available regardless:

- **[zoom](/docs/document-viewer/plugins/zoom)** ‚Äî toolbar gated by `features.zoom`
- **[rotate](/docs/document-viewer/plugins/rotate)** ‚Äî toolbar gated by `features.rotate`
- **[spread](/docs/document-viewer/plugins/spread)** ‚Äî toolbar gated by `features.spread`
- **[pan](/docs/document-viewer/plugins/pan)** ‚Äî toolbar gated by `features.pan`
- **[fullscreen](/docs/document-viewer/plugins/fullscreen)** ‚Äî toolbar gated by `features.fullscreen`

## Interaction

Both always registered:

- **[interaction-manager](/docs/document-viewer/plugins/interaction-manager)** ‚Äî always on (bridges pointer events for pan, selection, and annotation tools)
- **[selection](/docs/document-viewer/plugins/selection)** ‚Äî always registered; the `SelectionLayer` is gated by `features.selection`

## Discovery & navigation

Always registered; the listed flag toggles the corresponding UI:

- **[search](/docs/document-viewer/plugins/search)** ‚Äî search layer / popover gated by `features.search`
- **[bookmark](/docs/document-viewer/plugins/bookmark)** ‚Äî outline panel gated by `features.outline`
- **[thumbnail](/docs/document-viewer/plugins/thumbnail)** ‚Äî thumbnails panel gated by `features.thumbnails`

## Output

Always registered (both the Export and Print plugins):

- **[export](/docs/document-viewer/plugins/export)** ‚Äî Download / Print buttons gated by `features.download` / `features.print`; powers `saveAsCopy()`

## Editing (Phase 1)

Always registered:

- **[hotkeys](/docs/document-viewer/plugins/hotkeys)** ‚Äî Commands plugin; key bindings gated by `features.keyboardShortcuts`
- **[annotation](/docs/document-viewer/plugins/annotation)** ‚Äî annotation tools / toolbar gated by `features.annotations`
- **[history](/docs/document-viewer/plugins/history)** ‚Äî undo/redo surfaced via `features.undoRedo` (auto-on when annotations are enabled)

## Plugin registration pattern

Internally, `pluginRegistry.ts` builds the EmbedPDF registration list from the resolved `ViewerFeatures` and `FeatureConfig`. The order matters because some plugins depend on others (e.g. `annotation` needs `interaction-manager`, `selection`, and `history` to be registered first), so the registry emits them in dependency order.

All Phase 1 plugins are always registered ‚Äî this keeps the plugin composables safe to call unconditionally (Vue's `setup()` can't call `useZoom()` inside an `if`, and EmbedPDF composables throw when their plugin is missing). The `features` flags therefore control toolbar visibility and UX, not plugin presence. Only the Phase 2 plugins (`stamp`, `signature`, `redaction`, `form`, `attachment`) are conditionally registered.

`resolveFeatures` merges a partial `ViewerFeatures` over `DEFAULT_FEATURES`:

```ts
import { resolveFeatures } from '@meldui/vue'

const merged = resolveFeatures(userFeatures)
// merged is the effective Required<ViewerFeatures> (defaults filled in)
```
